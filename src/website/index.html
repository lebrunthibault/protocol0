What it does

A super flex keymapper… and more

Persistent hotkey mapping

Your shortcuts survive across sets — set them once, keep them forever.

Modifier support

Build real key combos with Ctrl, Alt and Shift.

Additional smart actions

Beyond basic mappings, use smart actions like loading a device or selecting a track by name — not by counting clicks.

Web UI to configure shortcuts

Map, edit and review everything from a clean browser interface.

HTTP API to control Live

Plug your own tools to trigger actions (Stream Deck, custom scripts)

Free, open source & extensible

MIT-spirited and built to be forked, scripted and extended (See Github repository for details).

Ableton vs Protocol 0

Two different key mappers

Ableton's keymapper is a basic, per-set, direct parameter mapping mode. Protocol 0 is a global and smart keymapper.

Ableton keymapper Protocol 0
Scope Mappings stored per set Mappings persist across all sets
Key combos Single key mapping Mappings including modifiers
Actions Direct parameter mapping Mappings include smart actions
Configuration Configurable in Live Configurable via a web interface
Setup Native Activate it with the Protocol 0 remote script
Nature Static Open source & extensible

Background story

Why Protocol 0 exists

I've been an Ableton user for more than ten years. I started using it around the same time I started coding. I eventually found out you could script Ableton with Python — first by using ClyphX user actions, and then remote scripting. It's been hard, because there isn't a lot of documentation! (The best reference I've found since is Structure Void's unofficial MIDI Remote Scripts docs.)

I love Ableton. It's a great piece of software, even though it has a few rough edges. One of them is the key mapper, which feels too basic and limited. I've been using custom shortcuts for a while now, and finally decided I wanted to share my work with the community!
